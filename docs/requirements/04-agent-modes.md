# 에이전트 모드

## 개요

시스템은 두 가지 에이전트 모드를 지원하며,
사용자가 설정 패널에서 실시간으로 전환할 수 있습니다.

## 모드 비교

| 항목 | 싱글 에이전트 | 멀티 에이전트 |
|------|--------------|--------------|
| 에이전트 수 | 1개 | 2개 (순차) |
| 파이프라인 | 요청 → 생성 → 응답 | 요청 → 생성 → 리뷰 → 응답 |
| 응답 속도 | 빠름 | 상대적으로 느림 |
| 품질 | 기본 | 리뷰를 거쳐 개선됨 |
| 용도 | 빠른 초안, 반복 실험 | 최종 품질 메시지 생성 |

## 싱글 에이전트 모드

하나의 Strands Agent가 사용자 요청을 받아 Amazon Bedrock LLM을 호출하고
결과를 직접 반환합니다. 간단하고 빠릅니다.

## 멀티 에이전트 모드

두 개의 에이전트가 순차적으로 동작합니다:

1. **생성 에이전트 (Agent1)**: 사용자 요청을 받아 초안 메시지를 생성합니다
2. **리뷰 에이전트 (Agent2)**: 초안을 검토하고 품질을 개선하여 최종 결과를 반환합니다

> ⚠️ Swarm이나 Graph 방식이 아닌, 순차적(sequential) 파이프라인입니다.

## 흐름 다이어그램

```mermaid
flowchart TD
    Request[사용자 요청] --> Check{에이전트 모드?}
    Check -->|Single| SA[단일 에이전트]
    SA --> LLM1[Amazon Bedrock]
    LLM1 --> R1[생성 결과]
    Check -->|Multi| GA[생성 에이전트 Agent1]
    GA --> LLM2[Amazon Bedrock]
    LLM2 --> Draft[초안 메시지]
    Draft --> RA[리뷰 에이전트 Agent2]
    RA --> LLM3[Amazon Bedrock]
    LLM3 --> R2[개선된 최종 결과]
    R1 --> Response[클라이언트 응답]
    R2 --> Response
```

## 모드 전환

- 플로팅 설정 패널에서 토글 스위치로 전환합니다
- 전환 시 기존 대화 기록은 유지됩니다
- 다음 요청부터 선택된 모드가 적용됩니다
