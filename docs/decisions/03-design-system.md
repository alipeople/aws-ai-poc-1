# ADR-03: 디자인 시스템

## 상태

승인됨

## 컨텍스트

POC에서 5개 테마를 지원해야 하며, 런타임에 테마 전환이 가능해야 합니다.
CSS-in-JS 라이브러리 의존 없이 순수 CSS 변수 기반 접근을 선택합니다.

## 결정

### 테마 목록

| 테마 | 키 | 특징 |
|------|-----|------|
| Sendon | `sendon` | 센드온 브랜드 컬러 기반, 기본 테마 |
| Toss | `toss` | 토스 스타일 블루 계열, 깔끔한 UI |
| Retro 8bit | `retro` | 픽셀 폰트, 레트로 컬러, 게임 느낌 |
| Dark | `dark` | 다크 배경, 눈 피로 감소 |
| Pastel | `pastel` | 부드러운 파스텔 톤, 밝은 분위기 |

### CSS 변수 접근법

`<html>` 태그의 `data-theme` 속성을 기반으로 테마를 전환합니다:

```css
/* 예시 구조 */
html[data-theme="sendon"] {
  --pri: #주요색상;
  --priL: #주요색상-밝은;
  --bg: #배경색;
  --surface: #표면색;
  --text: #텍스트색;
  --textSub: #보조텍스트색;
  --border: #테두리색;
  --accent: #강조색;
}
```

### 변수 네이밍 규칙

| 변수명 | 용도 |
|--------|------|
| `--pri` | Primary 색상 |
| `--priL` | Primary Light (밝은 변형) |
| `--bg` | 페이지 배경색 |
| `--surface` | 카드/패널 배경색 |
| `--text` | 기본 텍스트 색상 |
| `--textSub` | 보조 텍스트 색상 |
| `--border` | 테두리 색상 |
| `--accent` | 강조/액션 색상 |

### 플로팅 설정 패널

- 화면 우측에 고정된 플로팅 패널 제공
- 개발 시 테마, 에이전트 모드, LLM 모델을 실시간 전환 가능
- `SettingsContext`를 통해 전역 상태 관리
- 테마 전환 시 `document.documentElement.dataset.theme` 값을 변경

### 테마 적용 원리

1. `SettingsContext`에서 현재 테마 상태 관리
2. 테마 변경 시 `<html data-theme="...">` 속성 업데이트
3. CSS 변수가 자동으로 새 테마 값을 참조
4. 모든 컴포넌트가 별도 코드 변경 없이 즉시 반영

## 결과

- CSS 변수만으로 테마 전환 → 런타임 오버헤드 최소
- 새 테마 추가 시 CSS 변수 세트만 정의하면 됨
- JavaScript 번들 크기에 영향 없음
- 컴포넌트 코드에서 테마 로직 분리 (관심사 분리)
